# Engineering Taxonomy Update

This revision expands the reliability stack beyond prompt, context, harness, and situation engineering.

## Added disciplines
- Retrieval engineering
- Memory engineering
- Tool / function-calling engineering
- Protocol / interface engineering, including MCP
- Evaluation engineering
- Observability / provenance engineering
- Inference / serving engineering
- Guardrail / security engineering

## Organizing model
The disciplines are grouped into three interacting planes:
1. Semantic and cognitive plane
2. Action and runtime plane
3. Measurement and resource plane

Situation engineering remains distinct because it owns evidence applicability, active-world-state construction, uncertainty propagation, and the answer/retrieve/clarify/abstain/action gate.
